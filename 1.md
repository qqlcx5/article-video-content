
# Role
你是一位精通 Tauri + Vue3 + Rust 的全栈软件架构师，擅长处理本地数据持久化和状态管理。

# Context
我正在开发一个“抖音UP主视频管理”桌面应用。
- **技术栈**: Tauri (Rust) + Vue 3 (Script Setup) + Element Plus + TypeScript。
- **核心功能**: 管理从爬虫获取的视频数据，标记视频是否已使用（生成文稿），并检测视频是否在抖音端被删除。
- **数据流**:
  1. 用户定期导入/拖入一个新的 `crawler_result.json`（最新爬取数据）。
  2. 应用需要将新数据与本地存储的 `app_database.json` 进行合并。

# Data Structure
爬虫提供的原始 JSON 数据项 (`ICrawlerVideo`) 如下：
```json
{
  "id": "7545183817519074594",
  "title": "无标题",
  "href": "https://www.douyin.com/...",
  "thumbnail": "...",
  "likeCount": 0
}
```

但我需要在应用内部维护一个扩展模型 (`ILocalVideo`)，增加以下业务字段：
- `isUsed` (boolean): 是否已调用接口/生成文稿（这是我最看重的数据，不能因为更新JSON而丢失）。
- `localStatus` (string): 枚举值 'active' (正常) | 'lost' (已丢失/被删)。
- `lastSeen` (number): 时间戳，用于判断视频是否在最新列表中消失。

# Task
请基于“仓库模式 (Repository Pattern)”和“软删除 (Soft Delete)”策略，帮我实现核心的数据同步逻辑。

## 需求 1: TypeScript 类型定义
定义 `ICrawlerVideo` 和 `ILocalVideo` 接口。

## 需求 2: 核心同步算法 (Merge Logic)
请写一个 TypeScript 函数（或 Rust Command，如果你认为后端处理更好），名为 `syncVideoData`。
**逻辑要求：**
1. **输入**: 此时刻的爬虫数据列表 `latestVideos`。
2. **读取**: 读取本地数据库（如果为空则初始化）。
3. **遍历新数据**:
   - 如果 ID 不存在 -> **新增** (Status='active', isUsed=false, lastSeen=Now)。
   - 如果 ID 已存在 -> **更新** (更新 title, likeCount 等基础信息; 更新 lastSeen=Now; 如果之前是 'lost' 则恢复为 'active')。**注意：绝对不要覆盖 `isUsed` 状态！**
4. **检测丢失**:
   - 遍历本地所有数据。
   - 如果某视频的 `lastSeen` 早于本次更新时间 -> 标记 `localStatus` = 'lost' (代表视频丢失)。
5. **输出**: 返回合并后的完整列表，并保存回本地文件。

## 需求 3: Vue 组件逻辑
展示如何在 Vue 组件中使用这个逻辑：
- 导入 JSON 后调用同步函数。
- 在 Element Plus 表格中：
  - 如果 `localStatus === 'lost'`，行样式变灰/红色删除线，提示“可能已删除”。
  - 如果 `isUsed === true`，行透明度降低，显示“✅ 已使用”。
  - 提供一个按钮“生成文稿”，点击后调用 API 并将 `isUsed` 设为 `true`。

# Constraints
- 代码必须健壮，防止数据意外覆盖。
- 使用 TypeScript。
- 保持代码整洁，模块化。
